## Key Things I Found Interesting

**inferSchema=True is deceptively dangerous —**
I expected it to either work or throw an error on bad data.
What actually happens is worse — it works, but silently converts
rows it can't parse to null. You'd never know unless you checked.
Q14's live demo made this very concrete.

**null and empty string are completely different checks —**
`isNull()` won't catch `''` and `== ''` won't catch null.
This came up in Q12 and is easy to get wrong in practice.

**Spark's .filter() behaves like WHERE or HAVING depending on position —**
Same method, completely different behavior depending on whether
it comes before or after `.groupBy()`. No separate keyword like SQL.

**Immutability forces a cleaner coding style —**
Coming from pandas where you modify DataFrames in place, Spark's
immutability feels like a constraint at first. But it forces you to
think in terms of transformation chains, which makes pipelines
easier to debug and reason about.

**Q15 pipeline — the chained version and step-by-step version produce
identical results** because Spark's Catalyst optimizer converts both
into the same logical plan before executing anyway.

## How to Run

1. Make sure Java 11+ is installed (`java -version`)
2. Install PySpark:
   ```bash
   pip install pyspark
   ```
3. Open `spark_dataframe_operations.ipynb` in Jupyter
4. Run Cell 1 (SparkSession) first — confirm `PySpark version` prints cleanly
5. Run Cell 2 (dataset generation) — creates `raw_data/ecommerce_data.csv`
6. Run Cell 3 (load into Spark DataFrame) — confirms 1000 rows loaded
7. All 15 question cells can then be run top to bottom
